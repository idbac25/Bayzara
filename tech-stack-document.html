1. Platform Overview

Bayzara is a multi-tenant SaaS business management platform built specifically for Somali businesses. It handles invoicing, clients, vendors, inventory, purchases, leads (CRM), reports, and native Hormud EVC Plus payment integration. Multiple businesses can run on the same platform, each completely isolated from one another.

Architecture
Multi-Tenant SaaS
Each business gets its own slug-based workspace (e.g. /app/my-business). All data is isolated by business_id at the database level using Row Level Security.
Target Market
Somalia & East Africa
Built-in support for Somali Shilling, USD, EVC Plus mobile payments, and East Africa timezones. Default language is English with Somali context throughout.
Deployment
Cloud-Native
Fully serverless. No traditional servers to manage. Runs on Vercel (frontend + API) and Supabase (database + auth + storage + edge functions).
Codebase
Monorepo
Single repository on GitHub containing the Next.js app, Supabase migrations, and Supabase Edge Functions. One push deploys everything.

3. Backend — Server & API Layer

Bayzara has no traditional backend server. Instead it uses Next.js's built-in server capabilities and Supabase's managed infrastructure.

Next.js Server Components (Primary Data Layer)

The majority of data fetching happens in React Server Components — these run on Vercel's servers, query Supabase directly, and return fully rendered HTML to the browser. There is no separate REST API for page data.

Next.js API Routes (Action Layer)

Sensitive operations that cannot run in the browser are handled by API routes at /app/api/:

RouteMethodPurpose
/api/business/createPOSTCreates a new business, adds the owner as super_admin, creates default pipeline and document sequences. Uses service role key to bypass RLS on first membership insert.
/api/evc/connectPOSTAuthenticates with Hormud merchant API, fetches balance and account details, stores EVC connection.
/api/evc/activatePOSTActivates the EVC connection and creates a linked payment account.
/api/pdf/invoice/[id]GETRenders a print-ready HTML invoice page. Opening this URL and printing gives a PDF.
/api/auth/callbackGETOAuth callback handler for Google login (exchanges code for session).

Middleware

A Next.js middleware file runs on every request before the page loads. It handles:

  • Refreshing the user's Supabase session cookie on each request
  • Redirecting unauthenticated users away from protected routes (/app/*, /onboarding)
  • Redirecting already-logged-in users away from /login and /signup
  • Allowing public routes (/i/* public invoice links, /api/*) to pass through freely

profilesOne row per registered user. Stores name, email, phone, avatar, platform admin flag.Extends Supabase auth.users (1-to-1)

payment_recordsRecords of payments made against invoices — EVC, cash, bank transfer.Belongs to document and payment_account

Row Level Security (RLS)

Every table has RLS enabled. The central helper function is_business_member(business_id) is used across all policies — it checks whether the currently logged-in user has a record in business_users for the given business. No data from one business can ever be read or modified by another business's users.

Database Functions

FunctionPurpose
is_business_member(uuid)Returns true if the current user is a member of the given business. Used in all RLS policies.
get_next_document_number()Atomically generates the next sequential document number (e.g. INV-0042) using the document_sequences table. Thread-safe.
mark_overdue_invoices()Scans all sent invoices past their due date and updates their status to "overdue."

5. Authentication & Security

Authentication Provider

Supabase Auth handles all authentication. It manages user registration, login, password hashing, session tokens (JWTs), and OAuth. Bayzara supports:

Active
Email + Password
Standard signup and login. Email confirmation is optional (currently disabled for faster testing). Passwords are hashed by Supabase using bcrypt.
Configured
Google OAuth
One-click Google sign-in. Configured via Supabase Auth providers. Callback handled at /auth/callback.

Session Management

Sessions are managed using HTTP-only cookies via the @supabase/ssr package. This is more secure than localStorage (cookies cannot be accessed by JavaScript, preventing XSS attacks). The middleware refreshes the session on every request.

Role-Based Access

RoleScopePermissions
Platform AdminEntire platformAccess to /admin — can see all businesses, all users, platform revenue. Set via is_platform_admin flag on profiles table.
super_adminOne businessFull access to all features of their business. Can manage team members.
adminOne businessCan manage documents, clients, vendors, inventory. Cannot change billing or owner settings.
memberOne businessRead and create access. Cannot delete or change settings.

7. EVC Plus Integration (Hormud)

Bayzara has native, deep integration with Hormud EVC Plus — Somalia's most widely used mobile money system. This is a core differentiator of the platform.

Integration Architecture

Bayzara App
→
/api/evc/connect
→
Hormud Merchant API
→
EVC Transactions
→
Auto-Match Invoices

What the Integration Does

FeatureHow It Works
Merchant LoginServer-side API call to merchant.hormuud.com/api/account/login. Returns session token + cookie stored securely.
Balance SyncCurrent EVC wallet balance fetched and stored in evc_connections.current_balance. Updated every 60 seconds.
Transaction ImportUp to 50 new transactions fetched per sync cycle. Deduplicated by tran_id — never imported twice.
Auto-MatchingSender phone compared against client evc_phone field. If matched, linked to oldest open invoice.
Auto-RecordingIf amount matches invoice exactly → payment auto-recorded, invoice marked paid. No manual action needed.
Needs Review QueueUnmatched or ambiguous transactions are flagged and shown in the EVC Hub for manual assignment.

8. Background Jobs & Edge Functions

Long-running or scheduled tasks run as Supabase Edge Functions — serverless Deno functions deployed directly to Supabase's infrastructure, close to the database.

FunctionRuntimeSchedulePurpose
evc-syncDeno (TypeScript)Every 60 secondsPolls Hormud API for all active EVC connections, imports new transactions, auto-matches to clients and invoices, updates balances.
Deployment note: Edge Functions are separate from the Next.js app. They must be deployed to Supabase using supabase functions deploy evc-sync and scheduled using a cron trigger. This step is required for automatic EVC sync to activate.

Email Notifications (Configured)

Resend is integrated for transactional emails — invoice delivery, payment confirmations, team invitations. The API key is stored in environment variables. Email templates use Bayzara's brand colors.

11. Request Flow — End to End

Here is what happens technically when a user opens an invoice page:

#StepWhere it happens
1User navigates to /app/my-biz/invoicesBrowser
2Middleware runs — checks session cookie, refreshes JWT, verifies user is authenticatedVercel Edge Network
3[slug]/layout.tsx runs — fetches business record and user membership from Supabase using admin clientVercel Server
4invoices/page.tsx runs — queries documents table for all invoices in this businessVercel Server → Supabase PostgreSQL
5RLS policies on Supabase validate the query — only returns rows where business_id matches and user is a memberSupabase PostgreSQL
6Server component renders the full HTML with data and sends to browserVercel Server
7Browser renders the page — React hydrates interactive elements (search, filters, modals)Browser
8User creates an invoice — React Hook Form collects data, Supabase JS client sends INSERT to SupabaseBrowser → Supabase REST API
